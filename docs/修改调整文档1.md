## 一、地面视角绿色方块问题修复

### 问题描述
打开三维可视化的地面视角时，出现一层绿色的方块（图层）覆盖在视野上方，遮挡正常显示。

### 问题原因
1. **边界多边形**：遗产边界使用了 `extrudedHeight` 拉伸成立体形状，风险等级为 `safe` 时显示绿色
2. **植被多边形**：周边环境渲染中使用了绿色半透明多边形
3. 地面视角相机高度只有约 100 米，贴地的多边形遮挡视野

### 解决方案
地面视角时隐藏填充多边形，改用边界轮廓线显示：

```javascript
case 'ground':
  // 地面视角时：隐藏填充多边形，显示边界轮廓线
  this.boundaries.forEach(boundary => boundary.show = false)
  this.boundaryLines.forEach(line => line.show = this.showBoundaries)
  this.environmentEntities.forEach(entity => entity.show = false)
  break
```

## 二、地形切换逻辑颠倒问题修复

### 问题描述
左侧菜单勾选"地形"时不显示地形，不勾选时反而显示地形，用户体验混乱。

### 问题原因
`Cesium.createWorldTerrain()` 需要 Cesium Ion Token，但项目设置了 `Cesium.Ion.defaultAccessToken = undefined`：
- 勾选时调用 `createWorldTerrain()` 失败 → 地形不显示
- 不勾选时使用 `EllipsoidTerrainProvider()` → 正常显示

### 解决方案
通过控制影像层透明度实现地形显示切换：

```javascript
toggleTerrain() {
  const imageryLayers = this.viewer.imageryLayers
  if (this.showTerrain) {
    imageryLayers.get(0).alpha = 1.0  // 完全不透明，正常显示
  } else {
    imageryLayers.get(0).alpha = 0.3  // 半透明效果
  }
}
```

## 三、点击监测点位后相机锁定问题修复

### 问题描述
点击某个监测点位查看详细信息后，后续地图拖动只能以此为中心，无法自由移动。

### 问题原因
Cesium Viewer 默认启用了 `selectionIndicator` 和 `infoBox`，点击 entity 时会触发内置选中行为，导致相机"追踪"该实体。

### 解决方案
1. 初始化时禁用内置选中行为：
```javascript
this.viewer = new Cesium.Viewer('cesium-container', {
  selectionIndicator: false,  // 禁用选中指示器
  infoBox: false              // 禁用内置InfoBox
})
```

2. 点击处理器中清除追踪状态：
```javascript
if (this.viewer.trackedEntity) {
  this.viewer.trackedEntity = undefined
}
```

## 六、地面视角边界显示优化

### 问题描述
地面视角时，勾选遗产边界需要显示，但填充多边形会遮挡视野。

### 解决方案
为每个遗产地创建两种边界表示：
- **填充多边形**：用于太空/遗产地视角（高空俯瞰）
- **边界轮廓线**：用于地面视角（不遮挡视野）

新增数据属性：
```javascript
boundaryLines: []  // 地面视角时使用的边界轮廓线
```

渲染时同时创建两种：
```javascript
// 填充多边形
const boundary = this.viewer.entities.add({ polygon: {...} })
this.boundaries.push(boundary)

// 边界轮廓线
const boundaryLine = this.viewer.entities.add({ polyline: {...} })
boundaryLine.show = false  // 默认隐藏
this.boundaryLines.push(boundaryLine)
```

## 八、周边环境遮罩层功能

### 问题描述
周边环境勾选框需要实现：不勾选时，除了遗产地边界范围，其他地方均隐藏（显示遮罩）。

### 解决方案
新增遮罩层功能，使用 `PolygonHierarchy` 创建带"洞"的多边形：

1. 新增数据属性：
```javascript
areaMask: null  // 遮罩层实体
```

2. 新增 `renderAreaMask` 方法：
```javascript
// 收集所有遗产边界作为"洞"
const holes = []
this.areas.forEach(area => {
  const holePositions = points.map(p => Cesium.Cartesian3.fromDegrees(p[0], p[1]))
  holes.push(new Cesium.PolygonHierarchy(holePositions))
})

// 覆盖全球的大矩形 + 遗产地作为"洞"
this.areaMask = this.viewer.entities.add({
  polygon: {
    hierarchy: new Cesium.PolygonHierarchy(outerRing, holes),
    material: Cesium.Color.BLACK.withAlpha(0.6),
    height: 0
  }
})
```

3. 修改 `toggleEnvironment` 方法：
```javascript
if (this.showEnvironment) {
  // 勾选时：显示环境要素，隐藏遮罩
  this.areaMask.show = false
} else {
  // 不勾选时：隐藏环境要素，显示遮罩
  this.renderAreaMask()
}
```

## 十、添加遗产地自动生成边界框和遮罩

### 问题描述
添加遗产地后，三维可视化需要根据经纬度自动添加遗产地边界框、覆盖图层等，而不是依赖手动配置的 `boundaryPoints`。

### 解决方案
1. 新增 `generateBoundaryPoints` 方法：
```javascript
generateBoundaryPoints(centerLon, centerLat, radiusKm = 0.3) {
  const latOffset = radiusKm / 111
  const lonOffset = radiusKm / (111 * Math.cos(centerLat * Math.PI / 180))
  
  return [
    [centerLon - lonOffset, centerLat - latOffset],  // 西南
    [centerLon + lonOffset, centerLat - latOffset],  // 东南
    [centerLon + lonOffset, centerLat + latOffset],  // 东北
    [centerLon - lonOffset, centerLat + latOffset]   // 西北
  ]
}
```

2. 修改 `renderBoundary` 方法，自动生成边界点：
```javascript
if (area.boundaryPoints) {
  points = JSON.parse(area.boundaryPoints)
} else {
  points = this.generateBoundaryPoints(area.longitude, area.latitude)
}
```

3. 修改 `renderAreaMask` 方法，自动生成遮罩"洞"：
```javascript
if (!points || points.length < 3) {
  points = this.generateBoundaryPoints(area.longitude, area.latitude)
}
```

## 十一、二维地图新增区域不显示问题修复

### 问题描述
添加新区域后，二维地图可视化右侧菜单栏的快速定位列表没有显示新增的区域，地图上也没有生成点位显示。

### 问题原因
1. **边界点缺失**：`BaiduMapViewer.vue` 的 `renderAreaBoundary` 方法在开头检查 `if (!area.boundaryPoints) return`，导致没有 `boundaryPoints` 字段的新区域直接跳过不渲染
2. **添加区域表单缺少字段**：`Home.vue` 中 `areaForm` 只包含 `{ id, areaCode, areaName, description, latitude, longitude }`，没有 `boundaryPoints` 字段
3. **数据不自动刷新**：地图组件只在 `mounted` 时加载一次数据，添加区域后切换到地图页面不会自动更新

### 解决方案

#### 1. 修改 `BaiduMapViewer.vue` - 自动生成边界点

修改 `renderAreaBoundary` 方法，当没有 `boundaryPoints` 时根据经纬度自动生成：

```javascript
renderAreaBoundary(area) {
  // 必须有经纬度才能渲染
  if (!area.latitude || !area.longitude) return

  try {
    let points
    let center

    // 如果有边界点则使用，否则根据经纬度自动生成
    if (area.boundaryPoints) {
      points = JSON.parse(area.boundaryPoints)
    } else {
      points = this.generateBoundaryPoints(
        parseFloat(area.longitude),
        parseFloat(area.latitude)
      )
    }

    // ... 后续渲染逻辑
  } catch (error) {
    console.error('渲染区域边界失败', error)
  }
}
```

新增 `generateBoundaryPoints` 方法：

```javascript
// 根据中心经纬度自动生成矩形边界点
generateBoundaryPoints(centerLon, centerLat, radiusKm = 0.3) {
  // radiusKm: 边界半径，默认0.3公里
  // 1度纬度约等于111公里，经度随纬度变化
  const latOffset = radiusKm / 111
  const lonOffset = radiusKm / (111 * Math.cos(centerLat * Math.PI / 180))

  // 返回矩形的四个角点（顺时针）
  return [
    [centerLon - lonOffset, centerLat - latOffset], // 西南
    [centerLon + lonOffset, centerLat - latOffset], // 东南
    [centerLon + lonOffset, centerLat + latOffset], // 东北
    [centerLon - lonOffset, centerLat + latOffset]  // 西北
  ]
}
```

#### 2. 修改 `Home.vue` - 切换页面自动刷新数据

添加组件引用：

```javascript
// 地图组件引用，用于刷新数据
const baiduMapRef = ref(null)
const cesiumRef = ref(null)
```

修改模板，添加 ref 属性：

```html
<BaiduMapViewer ref="baiduMapRef" @view-trend="handleViewTrend" />
<CesiumViewer ref="cesiumRef" />
```

修改 `handleMenuClick` 函数，切换到地图页面时自动刷新数据：

```javascript
nextTick(() => {
  switch (menu) {
    case 'map':
      setTimeout(() => {
        window.dispatchEvent(new Event('resize'))
        // 刷新地图数据，显示新增的区域
        if (baiduMapRef.value?.loadData) {
          baiduMapRef.value.loadData()
        }
      }, 150)
      break
    case '3d':
      setTimeout(() => {
        window.dispatchEvent(new Event('resize'))
        // 刷新三维地图数据
        if (cesiumRef.value?.loadData) {
          cesiumRef.value.loadData()
        }
      }, 150)
      break
    // ... 其他 case
  }
})
```
